# 归并排序（Merge Sort）

## 一、算法概述

归并排序是一种基于**分治法（Divide and Conquer）**的经典排序算法。

核心思想：
1. **分**：将数组不断二分，直到每个子数组只有一个元素
2. **治**：将两个有序子数组合并成一个更大的有序数组
3. **合**：逐层向上合并，最终得到完整的有序数组

```
[5, 3, 8, 1, 9, 2]
    /           \
[5, 3, 8]     [1, 9, 2]
  /    \        /    \
[5,3]  [8]   [1,9]  [2]
 /  \    |     /  \   |
[5] [3] [8]  [1] [9] [2]
  \  /   |     \  /   |
 [3,5] [8]    [1,9] [2]
    \    /        \   /
  [3,5,8]       [1,2,9]
       \         /
    [1, 2, 3, 5, 8, 9]
```

## 二、核心代码

### 2.1 合并两个有序数组

```javascript
function merge(left, right) {
    const result = [];
    while (left.length && right.length) {
        if (left[0] <= right[0]) {
            result.push(left.shift());
        } else {
            result.push(right.shift());
        }
    }
    return result.concat(left, right);
}
```

### 2.2 归并排序主函数

```javascript
function mergeSort(arr) {
    if (arr.length <= 1) return arr;

    const mid = Math.floor(arr.length / 2);
    const left = mergeSort(arr.slice(0, mid));
    const right = mergeSort(arr.slice(mid));

    return merge(left, right);
}
```

## 三、复杂度分析

| 指标 | 复杂度 | 说明 |
|:---:|:---:|:---|
| **时间复杂度（最优）** | O(n log n) | 每次二分，每层合并 O(n) |
| **时间复杂度（最差）** | O(n log n) | 与最优情况相同，稳定 |
| **时间复杂度（平均）** | O(n log n) | 始终如此 |
| **空间复杂度** | O(n) | 需要额外数组存储合并结果 |
| **稳定性** | ✅ 稳定 | 相等元素保持原相对顺序 |

---

## 四、与其他排序算法对比

| 算法 | 平均时间 | 最坏时间 | 空间 | 稳定性 | 适用场景 |
|:---:|:---:|:---:|:---:|:---:|:---|
| **归并排序** | O(n log n) | O(n log n) | O(n) | ✅ 稳定 | 大数据量、需要稳定排序、链表排序 |
| 快速排序 | O(n log n) | O(n²) | O(log n) | ❌ 不稳定 | 通用场景，常数因子小 |
| 堆排序 | O(n log n) | O(n log n) | O(1) | ❌ 不稳定 | 内存受限场景 |
| 插入排序 | O(n²) | O(n²) | O(1) | ✅ 稳定 | 小数据量、近乎有序 |

---

## 五、特点总结

| 优点 | 缺点 |
|:---|:---|
| 时间复杂度稳定，始终 O(n log n) | 需要 O(n) 额外空间 |
| 稳定排序，相等元素顺序不变 | 不适合数据量极大的场景（空间开销） |
| 非常适合链表排序（无需随机访问） | 常数因子比快排略大 |
| 天然适合并行化（子问题独立） | - |

---

## 六、使用示例

```javascript
const arr = [5, 3, 8, 1, 9, 2];
const sorted = mergeSort(arr);
console.log(sorted);  // [1, 2, 3, 5, 8, 9]
```

---

## 七、适用场景

1. **数据量较大且需要稳定排序** —— 归并排序是首选
2. **链表排序** —— 不需要随机访问，归并排序非常高效
3. **外部排序** —— 数据无法全部载入内存时，归并排序可以分段处理
4. **并行计算环境** —— 左右子数组的排序完全独立，可并行执行
